# queries

LLM との QA，調査，計画検討の履歴を保存する場所．

query は思考過程やその時点の前提を残すための記録であり，最終的な概念知そのものではない．有用な発見は，後から `articles/`，`synthesis/`，`recipes/`，`decisions/` に蒸留する．

## 外部セッションの取り込み候補

別プロジェクトや別セッションで LLM と技術検討した内容は，まず query candidate として扱う．LLM は保存価値を判定し，必要に応じて `queries/` に保存するか，`incidents/`，`recipes/`，`decisions/`，`articles/` に蒸留する．

別セッションの最後に，以下のように依頼すると取り込みやすい．

```text
このセッションで議論した技術的要点を，
別の llm-wiki に取り込める形で要約して．
以下を含めて:

- 背景
- 対象 repo / branch / commit
- 調査したファイル
- 実行したコマンド
- 発生した問題
- 原因
- 採用した判断
- 次回使える手順
- 未解決事項
- secrets や機密値は含めない
```
