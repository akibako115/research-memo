# Sources — 執筆ガイドライン

source を書く際の粒度・構成・品質の指針．

## source の役割

- **raw の一次情報 1 件に対して source 1 件を作成する．**「この資料に何が書いてあるか」を忠実に要約・抽出する．LLM の解釈や意見は加えない．

## 記事の粒度

- **raw 1 件 → source 1 件**．1つの source が複数の raw をまとめることはしない．raw の内容が大きい場合でも分割せず，1 source にまとめる．

## 記事の構成パターン

### 1. フロントマター

```yaml
---
created: YYYY-MM-DD
updated: YYYY-MM-DD
raw: "raw/articles/YYYY-MM-DD_title.md"
---
```

### 2. 要旨（冒頭1-2段落）

- この資料が**何について**書かれているかを簡潔に述べる．
- 対象読者・前提知識・スコープがあれば記載する．

### 3. 主要ポイントの抽出

- raw の内容を構造化して整理する．
- 見出しは raw の構成に従うか，内容に即した具体的な名前を付ける．
- 数値・固有名詞・コード例など具体的な情報を落とさない．

### 4. 関連概念（末尾）

- この source から article に展開できそうな概念を `[[Concept_Name]]` で列挙する．

## 文体・表記

- 日本語で記述．句読点はコンマ(，)とピリオド(．)を使用．
- 関連概念は `[[Concept_Name]]` で参照（Obsidian wikilink）．
- 原文の用語・表現をできるだけ保持する．

## やること・やらないこと

| やること | やらないこと |
| --- | --- |
| raw の内容を忠実に要約する | LLM の解釈や意見を加える |
| 具体的な数値・コード例を残す | 抽象化して情報を落とす |
| 構造化して読みやすく整理する | raw をそのままコピーする |
| 関連する概念を wikilink で示す | 概念の説明を source 内で展開する |
| raw 1 件に対して source 1 件を作る | 複数の raw を 1 source にまとめる |
